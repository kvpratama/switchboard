## Project Overview

Switchboard is a self-hosted Telegram bot that answers natural-language questions about your Google Calendar using LangChain agents. It runs as a long-polling bot with persistent conversation memory via LangGraph checkpointers.

**Architecture:**
- **Entry point:** `main.py` — loads settings, builds the agent, starts the Telegram bot
- **Agent:** LangChain agent with `dynamic_prompt` middleware for timezone-aware date resolution
- **Tools:** Google Calendar API wrappers (list events, search events, get event details)
- **Memory:** LangGraph `AsyncSqliteSaver` checkpointer keyed by Telegram `chat_id`
- **Auth:** Google OAuth 2.0 (read-only calendar scope); Telegram user ID allowlist

---

## Tech Stack

- **Python 3.12+** — async-first runtime
- **LangChain** — agent framework with `create_agent` and `dynamic_prompt` middleware
- **LangGraph** — conversation memory via `AsyncSqliteSaver` checkpointer
- **python-telegram-bot** — Telegram bot API wrapper with long-polling
- **Google Calendar API** — read-only calendar access via OAuth 2.0
- **Pydantic** — settings management and validation
- **pytest + pytest-asyncio** — testing framework
- **Ruff** — linting and formatting
- **uv** — dependency management

## Code Conventions

### General
- Python **3.12+** minimum.
- **Async-first where applicable** — use `async def` for all I/O-bound functions (Google Calendar API calls, LLM calls, Telegram handlers).
- **Strict type hints** on every function signature, including return types. No bare `Any` unless unavoidable.
- **Docstrings on every function and class** using Google-style format.

### Environment Variables
- All secrets and configuration come from `.env` via `python-dotenv`.
- Access config only through the `Settings` object in `src/core/config.py` (Pydantic `BaseSettings`).
- Never hardcode secrets, API keys, or connection strings.

## Testing

- Test runner: `uv run pytest`
- Always follow Red–Green–Refactor TDD
- Use `pytest-asyncio` with `asyncio_mode = "auto"` in `pyproject.toml`.
- **Tests are co-located with their target** — a test for `nodes/retriever.py` lives at `nodes/retriever_test.py`, not in a separate `tests/` directory.
- Test files are named `<module>_test.py` and live in the same directory as the module they test.
- Mock all external services (Google Calendar API, OpenAI, Telegram) in unit tests — never hit live APIs in tests.
- Integration tests that require a live API are marked `@pytest.mark.integration` and skipped by default.
- Shared fixtures live in `conftest.py` at the project root (or a local `conftest.py` for directory-scoped fixtures).

## What NOT to Do

- Do not commit `.env` (it is in `.gitignore`)
- Do not commit `credentials.json` or `data/token.json` (they contain secrets)
- Do not instantiate the Google Calendar client outside `src/calendar/client.py`
- Do not use `pip install` — always use `uv add`
- Do not use bare `except:` — always catch specific exceptions
- Do not hardcode timezones — always use the user's configured timezone from settings
